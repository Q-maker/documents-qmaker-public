# Challenge Question Types

This section shows how the main QcmMaker question families look while playing in
Challenge mode.

In Challenge mode, each question is checked as soon as you answer or submit it.
QcmMaker then shows immediate feedback before moving to the next question.

## Question Families

| Family | What the learner does |
|---|---|
| [Selection](selection.md) | Choose one or more proposals. |
| [Typed answer](typed-answer.md) | Type a short answer. |
| [Fill in blanks](fill-in-blanks.md) | Fill expected answers in the displayed blanks or fields. |
| [Enumeration](enumeration.md) | Enter several expected answers as a list. |
| [Match columns](match-columns.md) | Pair each item from one column with the matching item in the other column. |
| [Put in order](put-in-order.md) | Reorder items into the expected sequence. |
| [Jumbled words](jumbled-words.md) | Rebuild a sentence from shuffled words. |

## Reading The Screens

Each page shows the progression that was available in the captured example:

| State | Meaning |
|---|---|
| Empty | The question before the learner answers. |
| Filled | The learner has entered or arranged an answer before submitting. |
| Feedback success | The answer was accepted immediately. |
| Feedback failure | The answer was rejected or skipped. |
| Feedback partial | Some expected answers were found, but the answer was not fully correct. |

After feedback, Challenge mode can pause briefly or move automatically to the
next question depending on the player settings. The bottom feedback band shows
the result while the question remains visible.
