# Exam Question Types

This section shows how the main QcmMaker question families look while playing in
Exam mode.

In Exam mode, you answer each question first. QcmMaker shows detailed feedback
only after the quiz is finished, when you open the correction review.

## Question Families

| Family | What the learner does |
|---|---|
| [Selection](selection.md) | Choose one or more proposals. |
| [Typed answer](typed-answer.md) | Type a short answer. |
| [Fill in blanks](fill-in-blanks.md) | Fill expected answers in the displayed blanks or fields. |
| [Enumeration](enumeration.md) | Enter several expected answers as a list. |
| [Match columns](match-columns.md) | Pair each item from one column with the matching item in the other column. |
| [Put in order](put-in-order.md) | Drag or reorder items into the expected sequence. |
| [Jumbled words](jumbled-words.md) | Rebuild a sentence from shuffled words. |

## Reading The Screens

Each page shows the same progression when available:

| State | Meaning |
|---|---|
| Empty | The question before the learner answers. |
| Filled | The learner has entered or selected an answer, but feedback is not shown yet. |
| Correction success | The answer was accepted. |
| Correction failure | The answer was rejected or incomplete. |
| Correction partial | Some expected answers were found, but the answer is not fully correct. |

The visible correction style can vary by question type. Green usually marks
accepted answers, red usually marks rejected or missing answers, and correction
comments may show the expected answer.
