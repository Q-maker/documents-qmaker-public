# Расширенный TXT-импорт

Используйте расширенный TXT-формат, когда стандартного формата недостаточно. Он позволяет указывать типы вопросов, прикреплять медиа по URL, добавлять метаданные и использовать Markdown- или HTML-содержимое.

Перед использованием этой страницы убедитесь, что вы понимаете [стандартный TXT-импорт](import-txt-standard-ru.md).

Полезно знать: расширенный формат мощнее, потому что перед импортом вопроса он передает QcmMaker инструкции. Используйте его, когда импорт должен сохранить намерение автора, а не только видимый текст.

## Расширенные типы вопросов

Добавьте тип вопроса в начало секции вопроса с помощью `@type`.

```text
@open
Каков результат (10 + 1) x 2?
*22
> 11 x 2 = 22.
```

Документированные метки типов включают:

| Тип | Для чего использовать |
|---|---|
| `auto` | Позволить QcmMaker определить тип вопроса. Можно опустить при редактировании. |
| `select_all` | Вопросы выбора с несколькими возможными ответами. Можно опустить при редактировании. |
| `open` | Короткий вводимый ответ. |
| `put_in_order` | Элементы, которые нужно расположить по порядку. |
| `fill_in_all_blank` | Заполнение нескольких пропусков в одном тексте. |
| `enumerate_all` | Перечисление всех ожидаемых ответов. |
| `match_all_column` | Сопоставление элементов между колонками. |
| `jumbled_words` | Восстановление предложения из перемешанных слов. |

Расширенный пример также показывает практические варианты, такие как `fill_in_each_blank`, `enumerate_each` и `match_each_column`.

## Добавить метаданные

Разместите метаданные в квадратных скобках после типа вопроса.

```text
@select_all[
question.image = https://example.com/question-picture.png
question.sound = https://example.com/question-audio.mp3
]
Соответствует ли этот звук этому животному?
*Да
-Нет
> Это звук животного.
```

Блок метаданных начинается с `[` и заканчивается `]`. Каждая строка метаданных использует такую структуру:

```text
target.property = value
```

Что делают метаданные: они описывают дополнительное поведение или ресурсы для вопроса. Например, они могут прикрепить изображение к условию, добавить звук или настроить способ интерпретации ответа.

## Добавить медиа к ответам

Используйте индексы вариантов, чтобы прикрепить медиа или параметры к конкретному ответу. Индексы начинаются с `0`, поэтому `proposals[0]` — это первый ответ, `proposals[1]` — второй и так далее.

```text
@put_in_order[
proposals[0].image = https://example.com/child.png
proposals[1].image = https://example.com/adult.png
]
Расположите эти лица от самого молодого к самому взрослому.
*
*
> Перетащите ответы в правильном порядке.
```

Распространенные метаданные вариантов включают:

| Метаданные | Назначение |
|---|---|
| `proposals[index].image` | Прикрепить изображение к ответу. |
| `proposals[index].sound` | Прикрепить звук к ответу. |
| `proposals[index].clue` | Добавить подсказку к ответу. |
| `proposals[index].case_sensitive` | Управлять тем, должен ли вводимый ответ учитывать регистр букв. |

Используйте URL-адреса, доступные для скачивания медиа. Data URI также могут использоваться, если приложение их поддерживает.

## Добавить медиа к комментариям

QcmMaker поддерживает одну секцию комментария для вопроса. Используйте `comments[0]` для метаданных комментария.

```text
@open[
comments[0].image = https://example.com/explanation.png
]
Назовите один цвет радуги.
*фиолетовый
*синий
*зеленый
> На изображении показана вся радуга.
```

## Форматировать содержимое с помощью Markdown или HTML

Оберните секцию вопроса в `$md{}`, если хотите использовать форматирование Markdown.

```text
$md{
@select_all
**Какая планета ближе всего к солнцу?**
*Меркурий
-Венера
-Земля
> Меркурий — первая планета Солнечной системы.
}
```

Оберните секцию вопроса в `$html{}`, если нужен HTML.

```text
$html{
@open
<h3>Назовите одну планету Солнечной системы</h3>
*Земля
*Марс
> Земля и Марс являются планетами.
}
```

## Расширенный контрольный список

- Начинайте с `@type`, когда хотите принудительно задать тип вопроса.
- Добавляйте `[...]` только тогда, когда нужны метаданные.
- Следите, чтобы индексы вариантов соответствовали порядку ответов.
- Используйте `comments[0]` для медиа комментария.
- Разделяйте секции вопросов одной пустой строкой.
- Предпочитайте кодировку UTF-8.

Если импорт не удался, сначала упростите файл: удалите метаданные, оставьте один вопрос, подтвердите, что базовая структура работает, затем добавляйте расширенные инструкции обратно по одной.

Вы можете начать с примера файла здесь: [demo_advanced-ru.txt](../../../../resources/qxt/demo_advanced-ru.txt).

Для более простого первого импорта вернитесь к [стандартному TXT-импорту](import-txt-standard-ru.md).
